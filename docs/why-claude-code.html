Platform Overview

Why Claude Code

More than an AI coding tool — an extensible platform for agent-driven workflows.

The Agentic Loop

The difference isn't the model — it's the harness. Claude Code runs three phases in a continuous loop: gather context, take action, verify results. When verification finds problems, it course-corrects and repeats. You can interrupt at any point.

Runs in the terminal, VS Code, JetBrains, or the desktop app. Same agent, same loop, multiple surfaces.

Composable Extension Primitives

Five building blocks shape how the agent thinks and acts. No plugin SDK, no proprietary config — text files in .claude/ that you commit to git.

CLAUDE.md

Your project's rulebook

Coding standards, conventions, constraints. Hierarchical — org, personal, project, local levels. More specific always wins.

Skills

Teachable expertise

Teach Claude your workflows: PR reviews, deployments, commit formats. Auto-activate on context match or invoke as slash commands.

Subagents

Delegate and conquer

Isolated workers for exploration, research, reviews. Three built-in types plus custom definitions with tool access control.

Hooks

Guardrails that never forget

Fire on events — before commands, after saves, at session end. Unlike instructions the AI might overlook, hooks execute every time.

MCP Servers

Plug in the outside world

Databases, GitHub, Sentry, browser automation, design tools. One command to add. Used like built-in capabilities.

How They Compose

The primitives aren't isolated features — they compose into layered workflows. Each handles a different concern.

1 Playwright MCP gives Claude browser automation. Without it, Claude can't interact with your web app at all.
2 A testing skill defines your team's test patterns — which pages to test, how to write assertions, what edge cases matter.
3 A subagent runs the tests in isolation, works through verbose output, and returns only the verdict.
4 A PostToolUse hook fires after every file Claude writes during the fix — runs the linter, every file, every time.
5 /test triggers this entire flow with one command.

Beyond Interactive Use

The primitives work in interactive sessions — but Claude Code goes further. Four capabilities take it beyond a tool you chat with.

Plugins

Bundle skills, subagents, hooks, and MCP configs into a single directory with a manifest. One install command wires everything up. Distribute via git or load locally.

Headless Mode

Run Claude non-interactively with claude -p. Pipe input, get structured JSON output, chain sessions. Use in shell scripts, CI/CD, batch processing.

Claude Agent SDK

The agentic loop as a Python and TypeScript library. Built-in tools work out of the box. Define hooks, subagents, and MCP connections programmatically.

Extensions

VS Code, JetBrains, Chrome, desktop app. The Claude in Chrome extension pairs with Claude Code as a built-in browser MCP server — see, click, and verify in the browser without leaving the session.

Claude Code is a platform for building agent-driven workflows. Text files define the behavior. Git carries it across your team. The agent loop runs the same whether you're in the terminal, VS Code, or a CI pipeline.
