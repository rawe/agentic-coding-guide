What Is a Skill?

A self-contained package of workflow instructions and optional resources that teaches Claude Code how to do something it cannot do natively.

Meet yt-extract

To make skills concrete, we'll use a real one throughout this page. yt-extract takes a YouTube URL and produces structured source files — metadata, transcript, and tags — without you writing any extraction code. You type one command, and Claude handles the rest: running a Python script, parsing the output, and updating your project's source index.

Let's open it up and see what's inside.

Anatomy of a Skill

A skill is a folder with a SKILL.md file and optional supporting resources. Here's the real yt-extract skill.

.claude/skills/yt-extract/ | |-- SKILL.md # Frontmatter (metadata) | # + body (instructions) | +-- scripts/ +-- yt-extract.py # 144-line extraction script # inline uv deps, runs as # subprocess

SKILL.md Frontmatter

YAML block with name, description, argument-hint, user-invocable. This is the only part always in context.

SKILL.md Body

Step-by-step instructions the agent follows when the skill is activated. References scripts and output locations explicitly.

scripts/

Executables that run as subprocesses. In general scripts/ is optional, but for yt-extract it's the heart of the skill — the Python script does all the heavy lifting. It declares its own dependencies via inline shebangs and never enters the context window.

references/

Optional. Templates, schemas, or examples that the agent reads selectively. Only loaded when the skill instructions say to read them.

But not all of this loads at once. Skills use progressive disclosure — Claude only pays for what it actually needs.

Progressive Disclosure

Skills are lazy-loaded in three layers. Only the description sits in context until Claude decides the skill is relevant. This is what makes it possible to install dozens of skills with near-zero token overhead.

Layer 1 — Metadata

Always Loaded

The YAML frontmatter — name and description — is always in context. The description is the primary triggering mechanism: Claude reads it to decide if this skill is relevant to the current task.

--- name: yt-extract description: Extract metadata and transcript from a YouTube video into the sources folder. argument-hint: [youtube-url] user-invocable: true ---
Context cost ~20 tokens
Layer 2 — Full Instructions

Loaded When Triggered

The SKILL.md body loads only after Claude decides the skill is relevant (or the user invokes it directly). Contains the step-by-step workflow, kept under 500 lines.

# YouTube Extraction Given a YouTube URL via `$ARGUMENTS`: 1. Run the extraction script: uv run .claude/skills/yt-extract/scripts/yt-extract.py '$ARGUMENTS' 2. Add relevant tags to the generated `meta.md` 3. Update `sources/index.md` with the new entry
Context cost ~100 tokens
Layer 3 — Bundled Resources

Loaded or Executed on Demand

scripts/ run as subprocesses — zero context cost. references/ are read selectively when Claude needs them. assets/ are used in output only. The heavy lifting stays outside the context window.

# /// script # dependencies = ["yt-dlp", "youtube-transcript-api"] # requires-python = ">=3.9" # /// """Extract metadata and transcript from a YouTube video.""" ... 144 lines total — runs as subprocess, not loaded into context
Context cost 0 tokens (subprocess)

Walk-through: yt-extract

yt-extract is a skill that takes a YouTube URL and produces structured source files — metadata, transcript, and tags — without you writing a single line of extraction code. Here's what happens when you use it, traced through all three disclosure layers.

1

Claude sees the description

At session start, only the frontmatter occupies context. Claude knows this skill exists and what it does — nothing more.

name: yt-extract description: Extract metadata and transcript from a YouTube video into the sources folder.
2

You trigger it

Type /yt-extract https://youtube.com/watch?v=... in Claude Code. The user-invocable: true flag makes the slash command available. Claude can also trigger it autonomously if a task matches the description.

3

Claude loads the full instructions

The SKILL.md body enters context. Claude now has a 3-step workflow to follow: run the script, tag the output, update the index.

1. Run the extraction script: uv run .claude/skills/yt-extract/scripts/yt-extract.py '$ARGUMENTS' 2. Add relevant tags to the generated meta.md 3. Update sources/index.md with the new entry
4

Claude executes the script

The 144-line Python script runs as a subprocess. It uses yt-dlp and youtube-transcript-api (declared as inline uv dependencies). The script fetches metadata and transcript, writes meta.md and transcript.md. Zero tokens consumed — only the output path returns to context.

# /// script # dependencies = ["yt-dlp", "youtube-transcript-api"] # requires-python = ">=3.9" # /// import argparse, re, sys from pathlib import Path import yt_dlp from youtube_transcript_api import YouTubeTranscriptApi ... extracts video metadata, transcript → writes to sources/

Where to Find Skills

Two paths to get skills into your project.

Recommended

skills.sh

Vercel's open agent skills ecosystem. Browse, search, one-command install.

  1. Browse skills at skills.sh
  2. Copy the install command
  3. Paste in terminal, choose CLI tool
  4. Pick project-level or global install
  5. Skill appears in .claude/skills/
Alternative

Plugin Marketplace

Built into Claude Code. Add marketplace repos, browse and install.

  1. Open Claude Code, go to /plugin
  2. Navigate to marketplaces
  3. Add a GitHub repository URL
  4. Browse available plugins
  5. Select with spacebar, press I to install

Creating Your Own

Here's how yt-extract was actually built — not by writing the SKILL.md by hand, but by collaborating with Claude Code and the /skill-creator meta-skill (installable from skills.sh).

Step 1

Build the Script

Guide Claude Code through writing the extraction logic — iterate on the Python script together until it works.

Step 2

Run /skill-creator

Point the skill creator at your working script. It generates the SKILL.md with frontmatter and instructions.

Step 3

Organize the Folder

Move the script into the scripts/ subfolder so the skill follows the standard layout.

Step 4

Test & Refine

Run /yt-extract on a real URL, check the output, and tweak the instructions if needed.

How yt-extract Was Made

  • Guided Claude Code to write a Python script that uses yt-dlp and youtube-transcript-api to extract video metadata and transcripts
  • Iterated until the script handled edge cases (missing transcripts, slug generation, timestamps)
  • Ran /skill-creator — it analyzed the script and generated the SKILL.md with the three-step workflow
  • Moved yt-extract.py into .claude/skills/yt-extract/scripts/

The Result

.claude/skills/yt-extract/ |-- SKILL.md # generated by /skill-creator +-- scripts/ +-- yt-extract.py # built collaboratively

The same skill you saw dissected above — created in one session, reusable forever.
