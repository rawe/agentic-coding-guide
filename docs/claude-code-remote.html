Remote Access

Claude Code Remote Control

Control Claude Code from your phone or any browser — send prompts, approve permissions, and run agents on remote servers without your laptop.

What Remote Control Is

claude rc opens a secure connection through Anthropic’s servers. Once active, you can send prompts from the Claude mobile app or web interface directly into your terminal session.

  • The session inherits your full local config: skills, MCP servers, plan mode, permission settings
  • Messages from the phone appear instantly in the terminal
  • Permissions can be approved from mobile when not using --dangerously-skip-permissions
Replaces the workaround: No more tmux + Tailscale + Terminus. One built-in command gives you mobile access to your full Claude Code environment.

Setup

Three ways to enable remote control, from one-off to always-on.

Per-Session

Quick Start

Run once to enable for the current session only.

claude rc

Or type /remote-control inside an active session.

Persistent

All Sessions

Enable once in config, active for every future session.

# Inside Claude Code: /config # Search "remote control" # Set to true
Remote Server

Always-On Agent

SSH into a VPS, run in tmux, detach. The session stays alive 24/7.

ssh your-server tmux claude --dangerously-skip-permissions /remote-control # Detach: Ctrl+B :detach
On the client side: Open the Claude app (mobile or web) → go to the Code tab → pick the session. Messages sent from the phone appear instantly in the terminal.

Three Security Strategies

Remote sessions need security boundaries. Three approaches, each with a different trade-off between convenience and control.

Isolation

Remote Server

Deploy Claude Code on a disposable VPS. Even with --dangerously-skip-permissions, damage is confined to the server.

  • Install MCP servers for 24/7 background tasks
  • Your laptop can be off entirely
  • If compromised, nuke and re-spin the server
  • Example: $3.49/mo Hetzner instance
Lockdown

Sandboxing

Configure settings.local.json to create a zero-write research environment.

  • URL allowlist (e.g., arxiv, Google Scholar)
  • Deny edit, write, and bash tools
  • No permission prompts needed
  • Claude can fetch but cannot modify anything
Filtering

Proxy

A local HTTP proxy filters network access at the domain level while keeping bash and curl available.

  • Deny WebFetch and WebSearch (they bypass the proxy)
  • Set httpProxyPort in settings
  • Node.js proxy with domain allowlist
  • api.anthropic.com must be allowed
Strategy Network Control File System Best For
Remote Server Full (isolated machine) Full access on server Always-on agents, disposable environments
Sandboxing URL allowlist Read-only Research-only agents, zero-write environments
Proxy Domain-level filtering Full access Agents that need bash but restricted network

Sandbox Configuration Example

A minimal settings.local.json that creates a locked-down research agent.

// .claude/settings.local.json { "sandbox": { "enabled": true }, "permissions": { "defaultMode": "dontAsk" }, "allowedUrls": [ "arxiv.org", "scholar.google.com", "pubmed.ncbi.nlm.nih.gov" ], "deny": ["edit", "write", "bash"] }
Result: Claude can read files and fetch from allowed URLs, but cannot modify your file system or run shell commands. No permission prompts appear because defaultMode: "dontAsk" auto-denies anything not explicitly allowed.

Remote Control vs OpenClaw

Remote control is passive (you prompt it). OpenClaw is proactive (it can schedule tasks and push notifications).

Remote Control

  • Interaction: You send prompts, Claude executes
  • Setup: Built-in — claude rc
  • Config: Inherits full local environment
  • Scheduling: None (manual prompts only)
  • Notifications: Not yet supported

OpenClaw

  • Interaction: Proactive — scheduled tasks, auto-triggers
  • Setup: Separate tool installation
  • Config: Independent configuration
  • Scheduling: Cron-like scheduled tasks
  • Notifications: Push notifications to phone
Bottom line: For on-demand mobile access to your Claude Code environment (skills, MCP servers, project context), remote control already covers most use cases. Anthropic is expected to add hooks for phone notifications during remote sessions, which would close most of the gap.
